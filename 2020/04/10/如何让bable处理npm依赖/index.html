怎么让bable处理npm依赖

#cli3   #js   #vue   

通常引用了一个npm依赖，我们是不会管它怎么实现的，都是直接使用的，因为通常都是做好兼容的，但是有些包你还是要经过bable处理一下，为了兼容ie。但是大家都知道，vue-cli编译都是过滤node_moudles的，如果你引用了一些没有经过es6转es5的依赖包，往往在ie你是打不开的，弹出各种错误，那么这种情况该如何处理？这里以resize-detector为例，这是一个监听dom大小变化的npm依赖

vue-cli2：

&#123;
  test: /\.js$/,
  loader: 'babel-loader',
  include: [resolve('src'), resolve('test')]
  // 修改一下下
  include: [
    resolve('src'),
    resolve('test'),
    resolve('node_modules/vue-echarts'),
    resolve('node_modules/resize-detector')
  ]
&#125;

这里直接不用exclude

vue-cli3

// vue.config.js
module.exports = &#123;
  transpileDependencies: [
    'resize-detector'
  ]
&#125;

在vue.config里面加这个选项
